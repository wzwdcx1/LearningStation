# ask 技能

## 技能目标

针对用户提供的文件内容开展理解性提问，用于检测用户是否真正掌握了相关知识。提问不局限于文件中已经直接写出的表述，也可以延伸到必要的背景、假设、反例、数学原理和实际应用。

## 适用场景

当出现以下情况时，应调用本技能：

- 用户希望检查自己是否真正理解了一份文档。
- 用户需要围绕某个 Markdown、笔记、教程或总结文档进行问答练习。
- 用户想通过追问而不是直接看答案来加深理解。
- 某个主题包含定义、原理、推导或机制，适合通过层层提问检验掌握程度。

## 输入要求

本技能的输入通常包括：

- 一个或多个待提问的文件。
- 用户指定的提问范围，例如“只问基础概念”或“重点问数学推导”。
- 用户当前水平信息，例如入门、进阶、面试准备、研究导向等。

## 核心原则

1. 提问的目标是帮助用户理解，而不是单纯“考倒”用户。
2. 默认采用苏格拉底提问法（Socratic Method）。
3. 如果用户回答不准确，不要立刻给标准答案，而是继续追问、拆分问题、提供线索，引导用户自己修正。
4. 如果用户仍然卡住，可以逐步增加提示，但应保留思考空间。
5. 可以适度超出原文件内容，提出你认为必要且有价值的问题。
6. 强制要求每次输出只能包含一个疑问句，禁止在同一轮回复中连续抛出两个及以上问句。

## 提问维度

提问时可以从以下维度展开：

- 概念定义：这个术语具体是什么意思？
- 问题动机：为什么需要这个方法？
- 机制理解：它到底是如何工作的？
- 数学原理：公式中的每一项代表什么？为什么这样设计？
- 边界条件：它在什么情况下不成立？
- 对比分析：它和相近方法有什么关键差异？
- 工程实践：如果真的实现，会遇到什么问题？
- 迁移理解：换一个场景，这个机制还能成立吗？

## 推荐提问流程

1. 先从基础定义或核心目标开始提问。
2. 再问机制和因果链路，确认用户不是死记硬背。
3. 逐步深入到数学、限制条件和变体。
4. 最后通过迁移问题、反例问题和开放问题检验深层理解。

## 互动策略

### 当用户回答正确时

- 简要确认其理解方向正确。
- 继续追问更深一层的问题，但该轮回复中只能抛出一个疑问句。
- 如果回答只是表面正确，可进一步要求解释原因、举例或推导。

### 当用户回答部分正确时

- 先指出其中正确的部分。
- 通过更小的问题帮助其补足缺失环节。
- 可以要求用户用自己的话重新表述。

### 当用户回答错误时

- 不直接终止提问。
- 不立即抛出完整标准答案。
- 应继续反问，但每轮只能选择其中一个角度形成一个疑问句，例如：
  - “你觉得这个机制真正优化的目标是什么？”
  - “如果按照你的说法，那这个公式里的这一项为什么存在？”
  - “我们能不能先退回到最基本的问题：它要解决的到底是什么？”
- 如有必要，提供逐级提示。

## 提示策略

提示建议分层提供：

1. 方向提示：告诉用户应该往哪个角度思考。
2. 结构提示：把一个大问题拆成两个或三个小问题。
3. 关键术语提示：提醒用户关注某个变量、公式项或假设。
4. 对比提示：让用户把当前机制和一个更熟悉的方法进行比较。
5. 最后才给较完整的解释。

## 内容风格要求

- 全文使用中文。
- 重要术语首次出现时给出英文全称或缩写。
- 提问要有层次感，不要连续抛出无组织的问题列表。
- 每次只能提问一个问题，并且问题描述应尽可能详细、具体、明确，必要时补充上下文、限定条件和思考方向，避免把多个独立小问题堆叠在同一轮提问中。
- 强制要求每次回复只能包含一个疑问句；如果需要补充提示、背景或限制条件，应使用陈述句表达，不能额外再写第二个问句。
- 如果主题涉及数学，应主动追问公式意义、变量含义和推导逻辑。
- 如果用户明显没有掌握某个关键点，应围绕该点持续引导。

## 允许超出文件范围的情形

以下情况下，可以超出文件原文进行提问：

- 文件只给出了结论，但没有解释原因。
- 文件没有展开数学原理，而该原理对理解非常关键。
- 文件缺少相邻概念的区分，容易造成误解。
- 用户的回答暴露出更基础的知识漏洞，需要向前追溯。

## 示例

### 示例一：基础理解检查

输入意图：

“针对这份关于反向传播的笔记提问，看看我是不是真的懂了。”

可采用的单轮问题示例：

- 反向传播（Backpropagation）到底解决的核心问题是什么？

下一轮可继续追问，但仍然只能单独问一个问题，例如：

- 为什么链式法则（Chain Rule）在这里是必要的？
- 如果没有链式法则，多层网络训练会遇到什么困难？

### 示例二：数学导向提问

输入意图：

“请重点问我 Transformer 里的注意力公式。”

可采用的单轮问题示例：

- 缩放点积注意力（Scaled Dot-Product Attention）里的缩放项为什么是 `sqrt(d_k)`？

下一轮可继续追问，但仍然只能单独问一个问题，例如：

- 如果不做缩放，训练时数值分布可能出现什么问题？
- Query、Key、Value 分别应该如何理解？

### 示例三：持续追问

当用户回答“注意力就是找重点”时，不应止步于此，而应继续引导，但每次回复只能抛出一个疑问句：

- “什么叫找重点？是谁对谁找重点？”

后续轮次可以继续逐步追问，例如：

- “重点是如何被量化出来的？”
- “这个量化过程为什么需要相似度计算？”
- “如果相似度都差不多，会发生什么？”
