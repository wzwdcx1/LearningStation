# continue 技能

## 技能目标

针对已经归档到 `archives` 文件夹中的学习内容，自动读取对应归档文档，并在其基础上继续展开该主题的讨论、补充、追问与深化，避免每次都从零重新建立上下文。

## 适用场景

当出现以下情况时，应优先调用本技能：

- 用户希望继续某个已经归档过的学习主题。
- 用户提到“接着上次”“继续之前那份归档”“基于 archives 里的内容继续讲”等意图。
- 用户希望围绕某份归档文档继续补充机制、数学推导、案例、对比分析或后续问题。
- 用户已经完成一轮学习归档，现在想在原有基础上继续深入。
- 用户没有完整重复背景，但明确表示要延续某个历史学习主题。

## 输入要求

本技能的输入通常包括但不限于以下信息：

- 用户指定的归档文件名。
- 用户给出的归档文件路径。
- 用户口头描述的主题名称，例如“继续 Transformer 那部分”。
- 用户希望继续展开的方向，例如“重点讲数学推导”“继续比较和 RNN 的区别”“继续给例子”。

## 核心职责

本技能被调用后，应完成以下工作：

1. 优先在 `archives` 文件夹中定位与用户意图最匹配的归档文档。
2. 自动读取目标归档内容，而不是要求用户重复粘贴历史内容。
3. 先简要识别归档中的已有结论、已讨论范围与未完成问题。
4. 基于原文脉络继续展开，而不是脱离上下文另起一套泛泛讲解。
5. 如果用户指定了继续方向，则优先沿该方向深入。
6. 如果用户没有指定方向，则优先围绕文档中的“待继续探索的问题”、薄弱环节或尚未展开的机制继续推进。

## 归档定位策略

当用户没有提供精确路径时，应按以下顺序定位目标文件：

1. 先在 `archives` 中查找文件名直接匹配的文档。
2. 如果没有完全匹配，则根据主题关键词寻找最相关的归档文件。
3. 如果存在多个候选文件，应先向用户澄清要继续哪一份，而不是擅自选择。
4. 如果未找到合适归档，应明确告知用户当前没有找到对应文档，并建议用户提供文件名、路径或更明确的主题关键词。

## 继续展开的优先顺序

在继续讲解时，优先遵循以下顺序组织内容：

1. 这个主题原本在解决什么问题。
2. 归档中已经讲清楚了什么。
3. 还有哪些关键机制、数学关系、边界条件或工程实现没有展开。
4. 本轮继续内容与原有归档之间是什么关系。
5. 继续学习后，应沉淀哪些新的结论、例子或待研究问题。

## 输出要求

继续展开学习内容时，输出应尽量满足以下要求：

- 不重复整份归档，而是先用简短内容承接已有上下文。
- 明确指出本轮是在上一份归档的哪个基础上继续推进。
- 如果适合，可以先总结“上次讲到哪里”，再进入新的展开内容。
- 讲解仍应优先关注概念、机制、数学原理、局限性与工程实现之间的联系。
- 当用户表现出理解不稳固时，可适度切换为引导式追问，而不是直接给结论。
- 若本轮讨论积累了足够多的新内容，应主动建议再次归档。

## 默认工作流程

1. 识别用户要继续的是哪一个历史主题。
2. 在 `archives` 中定位最匹配的归档文件。
3. 读取文档并提取当前主题的核心内容、已覆盖范围和待展开问题。
4. 结合用户本轮问题，决定继续展开的切入点。
5. 按“承接已有内容 -> 深化新问题 -> 总结新增理解 -> 提示后续方向”的顺序输出。
6. 如有必要，建议将新增讨论继续更新回归档文档。

## 内容风格要求

- 全文使用中文。
- 重要术语首次出现时，采用“中文（English, Abbreviation）”形式。
- 优先延续历史归档中的话题脉络和概念层次。
- 对复杂主题，优先按照“直觉 -> 机制 -> 数学 -> 工程 -> 局限”的顺序逐步展开。
- 避免脱离归档内容直接输出模板化百科式说明。
- 如果归档本身已经包含“待继续探索的问题”，应优先利用这些问题作为继续切入点。

## 与其他技能的关系

- 当目标是把当前讨论整理沉淀为文档时，应优先考虑 `archive` 技能。
- 当目标是围绕已有文档检验理解、连续追问时，可在继续阅读归档后配合 `ask` 技能的提问风格。
- 当目标是把归档中的抽象机制转换为示例代码时，可在继续上下文后配合 `code` 技能的表达方式。

## 示例

### 示例一：用户指定主题名

输入意图：

“继续我们之前归档在 archives 里的 Transformer 学习内容。”

输出行为：

- 在 `archives` 中寻找与 Transformer 相关的归档文档。
- 读取目标文件并先简要说明上次已经讨论到哪些内容。
- 继续展开尚未充分讨论的部分，例如多头注意力（Multi-Head Attention）、位置编码（Positional Encoding）或训练稳定性。

### 示例二：用户指定文件路径

输入意图：

“基于 `archives/强化学习-week1.md` 继续往下讲。”

输出行为：

- 直接读取该文件。
- 从文档已有结构出发，继续展开策略梯度（Policy Gradient）、价值函数（Value Function）或探索与利用（Exploration vs Exploitation）等后续内容。

### 示例三：用户只说继续上次内容

输入意图：

“继续上次归档的那部分世界模型内容，这次重点讲数学。”

输出行为：

- 先在 `archives` 中查找与世界模型（World Model）相关的归档文件。
- 如果有唯一高相关文档，则读取后继续。
- 如果有多个候选，则先询问用户要继续哪一份。
- 本轮优先围绕状态表示、目标函数、预测误差和规划关系展开数学层面的说明。
